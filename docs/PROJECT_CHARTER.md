# 项目章程 — 图书借阅管理系统

**版本：** 1.0
**日期：** 2026-03-05
**项目经理：** AI PM

---

## 一、项目背景与核心目标

解决图书馆人工记录借阅信息效率低、库存不实时更新、超期未还难追踪的问题。
提供一套 Web 界面的图书借阅管理系统，供管理员统一操作。

---

## 二、目标用户

| 角色 | 职责 |
|------|------|
| 图书管理员 | 图书录入/修改/删除、读者注册、处理借还操作、查看逾期列表 |

> 读者无自助操作端，所有操作由管理员代为执行。

---

## 三、交付范围（本期实现）

### 3.1 图书管理模块
- 新增图书：录入 ISBN、书名、作者、出版社、库存数量
- 批量导入：上传 CSV 文件批量导入图书（ISBN 重复时更新库存，新 ISBN 则新增）
- 图书查询：支持书名、作者、ISBN 关键字查询
- 修改/删除：更新图书信息；有未还记录时禁止删除

### 3.2 读者管理模块
- 读者注册：记录读者 ID、姓名、联系方式、最大借书数量（默认 5 本）
- 借阅权限校验：
  - 当前借书数量已达上限 → 禁止借书
  - 存在超期记录且处于 30 天禁借期内 → 禁止借书

### 3.3 借还管理模块
- 借书：输入读者 ID + 图书 ISBN，生成借书记录，扣减库存
- 还书：输入图书 ISBN（+ 读者 ID 确认），更新还书时间，恢复库存，触发超期处罚判断
- 逾期列表：管理员 Web 界面展示当前所有逾期未还记录

## 四、明确不做（本期排除）

- 图书预约功能
- 读者自助注册/自助查询
- 逾期短信/邮件通知
- 罚款金额计算与收缴
- 多馆联合借阅

---

## 五、核心业务规则

| # | 规则 | 说明 |
|---|------|------|
| R1 | 借阅期限 | 30 天，到期日 = 借出日期 + 30 天 |
| R2 | 超期处罚 | 还书时若已超期，对该读者施加 30 天禁借处罚（从还书日起算） |
| R3 | 禁借期校验 | 借书时检查读者是否在禁借期内，是则拒绝并提示禁借截止日期 |
| R4 | 借书上限 | 每位读者最多同时借 N 本（默认 5，管理员可修改） |
| R5 | 库存保护 | 库存为 0 时禁止借出 |
| R6 | 删除保护 | 图书存在未还记录时禁止删除 |
| R7 | 逾期定义 | 当前日期 > 应还日期 且 还书时间为空 |
