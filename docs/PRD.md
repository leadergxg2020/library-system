# 需求规格说明书 - 图书借阅管理系统

> 版本：1.0 | 日期：2026-03-05 | 状态：已确认

---

## 一、项目背景

### 1.1 行业背景与现状

图书馆日常运营依赖于图书借阅记录的准确性和实时性。传统管理方式（纸质台账或 Excel 表格）在馆藏规模增长后，面临数据分散、查找效率低、库存状态不实时、超期追踪困难等问题。管理员需要花费大量时间在手工核对工作上，极易出错，影响读者体验和馆务效率。

### 1.2 核心问题

本项目聚焦解决以下三个核心痛点：

1. **效率低下**：人工记录借还信息耗时、易出错，借书/还书操作需翻阅纸质记录。
2. **库存不实时**：图书借出后库存未同步更新，导致管理员无法准确掌握可借数量。
3. **超期追踪困难**：无法快速识别哪些图书已逾期未还，也无法对逾期读者进行规则约束。

### 1.3 项目目标与成功指标

**项目目标**：交付一套 Web 界面的图书借阅管理系统，供图书管理员统一操作，实现图书录入、读者注册、借还管理、逾期监控的数字化管理。

**成功指标**：

| 指标 | 目标值 |
|------|--------|
| 借书操作完成时间 | 单次操作 ≤ 30 秒 |
| 库存数据实时性 | 借/还操作后立即反映，延迟 ≤ 1 秒 |
| 逾期记录可查性 | 管理员一次点击即可看到全部逾期未还记录 |
| 系统可用性 | 本地部署，正常使用期间无崩溃 |

## 二、目标用户

### 2.1 用户角色定义

| 角色 | 描述 | 主要操作 |
|------|------|----------|
| 图书管理员 | 图书馆工作人员，系统唯一操作主体 | 图书录入/修改/删除、读者注册、处理借还操作、查看逾期列表 |

**说明**：本系统无读者自助操作端。所有操作（包括代表读者借书、还书）由图书管理员在 Web 界面完成。系统不设读者登录入口。

### 2.2 用户旅程地图（管理员完成借书操作的完整链路）

```
读者到柜台申请借书
       ↓
管理员在系统中输入读者 ID
       ↓
系统显示读者信息（姓名、当前借阅数、是否在禁借期）
       ↓
管理员确认读者状态正常（未超上限、不在禁借期）
       ↓
管理员输入图书 ISBN
       ↓
系统显示图书信息（书名、作者、当前库存）
       ↓
管理员确认库存充足，点击"确认借出"
       ↓
系统生成借阅记录（借出日期、应还日期 = 借出日期 + 30天）
系统库存数量 -1
       ↓
管理员告知读者应还日期，操作完成
```

## 四、功能详细说明

### 4.1 图书管理模块

#### 4.1.1 新增图书（单条录入）

**功能概述**

管理员通过 Web 表单逐条录入图书基本信息，系统校验 ISBN 唯一性后将图书信息持久化到数据库。该功能用于新到馆图书的初始登记。

**用户故事**

As a 图书管理员, I want to 通过表单录入单本图书的基本信息, So that 新到馆图书能被系统管理，并可参与借阅流程。

**业务规则**

| 编号 | 规则 |
|------|------|
| BK-01 | ISBN 在系统中必须唯一，重复提交时系统拒绝并提示"该 ISBN 已存在，如需增加库存请使用修改功能" |
| BK-02 | 库存数量必须为正整数（≥ 1） |
| BK-03 | 所有必填字段（ISBN、书名、作者、出版社、库存数量）均不允许为空提交 |
| BK-04 | 新增成功后，图书状态为"在库"，可参与借阅 |

**操作流程**

1. 管理员点击"新增图书"按钮，系统展示新增图书表单。
2. 管理员填写 ISBN、书名、作者、出版社、库存数量。
3. 管理员点击"提交"按钮。
4. 系统校验所有字段：
   - 若有字段为空，在对应字段下方显示错误提示，表单不提交。
   - 若 ISBN 格式不符合规则，在 ISBN 字段下方显示格式错误提示，表单不提交。
   - 若 ISBN 已存在，显示冲突提示，表单不提交。
5. 校验通过后，系统将图书信息写入数据库。
6. 系统显示成功提示"图书添加成功"，表单重置为空，等待继续录入。

**状态说明**

| 状态 | 触发条件 | 系统展示内容 |
|------|----------|-------------|
| 空状态 | 管理员初次打开新增图书表单 | 所有字段为空，无任何提示信息，"提交"按钮可点击 |
| 加载状态 | 管理员点击"提交"后，系统处理中 | "提交"按钮变为不可点击状态，显示"提交中..."文字，防止重复提交 |
| 成功状态 | 图书成功写入数据库 | 页面顶部显示绿色成功提示："图书添加成功"，表单所有字段清空 |
| 错误状态 | 字段校验失败或 ISBN 重复 | 在对应字段下方显示红色错误文案（见字段说明），表单不清空，保留已填内容 |
| 边界状态 | ISBN 为系统已存在值 | 在 ISBN 字段下方显示："该 ISBN 已存在，如需增加库存请使用修改功能" |

**字段说明**

| 字段名 | 显示名 | 类型 | 必填 | 校验规则 | 错误提示文案 |
|--------|--------|------|------|----------|-------------|
| isbn | ISBN | 文本 | 是 | 10位或13位纯数字，不含横线或空格 | "ISBN 格式不正确，请输入10位或13位纯数字" |
| title | 书名 | 文本 | 是 | 1～100 个字符 | "书名不能为空" / "书名不能超过100个字符" |
| author | 作者 | 文本 | 是 | 1～50 个字符 | "作者不能为空" / "作者不能超过50个字符" |
| publisher | 出版社 | 文本 | 是 | 1～100 个字符 | "出版社不能为空" / "出版社不能超过100个字符" |
| quantity | 库存数量 | 整数 | 是 | 正整数，范围 1～9999 | "库存数量不能为空" / "库存数量必须为1到9999之间的正整数" |

**验收标准**

- [ ] 提交所有字段合法且 ISBN 不重复时，图书记录写入数据库，数据库中 quantity 字段值与输入一致。
- [ ] ISBN 字段输入非纯数字（含字母、横线、空格）时，提交被阻止，错误提示显示在 ISBN 字段下方。
- [ ] ISBN 字段输入不足10位或11-12位或超过13位时，提交被阻止，错误提示显示在 ISBN 字段下方。
- [ ] 任何必填字段为空时，提交被阻止，错误提示显示在对应字段下方。
- [ ] 提交已存在的 ISBN 时，提交被阻止，页面提示"该 ISBN 已存在，如需增加库存请使用修改功能"。
- [ ] 提交成功后，表单所有字段清空，页面显示绿色成功提示。
- [ ] 提交过程中"提交"按钮不可重复点击。

#### 4.1.2 批量导入图书（CSV）

**功能概述**

管理员上传符合格式要求的 CSV 文件，系统逐行解析并按规则处理：ISBN 已存在则累加库存，ISBN 不存在则新增记录。导入完成后展示详细的处理结果报告，包括成功数量和每行失败原因。

**用户故事**

As a 图书管理员, I want to 上传 CSV 文件批量导入图书数据, So that 新到馆的大批量图书能快速入库，无需逐条手动录入。

**业务规则**

| 编号 | 规则 |
|------|------|
| BK-CSV-01 | CSV 文件第一行必须为表头，表头列名严格匹配（区分大小写）：isbn, title, author, publisher, quantity |
| BK-CSV-02 | CSV 文件编码必须为 UTF-8 |
| BK-CSV-03 | 文件扩展名必须为 `.csv`，其他文件类型拒绝上传 |
| BK-CSV-04 | 单次上传文件大小不超过 10MB |
| BK-CSV-05 | ISBN 在数据库中已存在：quantity 字段值累加到现有 quantity，书名/作者/出版社等字段保持不变 |
| BK-CSV-06 | ISBN 在数据库中不存在：新增图书记录，所有字段取自 CSV 行内容 |
| BK-CSV-07 | 单行数据存在格式错误时（字段缺失、字段值校验失败），跳过该行，记录错误，继续处理后续行 |
| BK-CSV-08 | 同一 CSV 文件中出现重复的 ISBN：所有重复行按顺序依次处理，每行 quantity 均累加（不视为错误） |
| BK-CSV-09 | 导入完成后展示结果摘要（不可关闭，需管理员主动点击"确认"才能进行下一操作） |

**CSV 文件格式说明**

第一行为表头，从第二行起为数据行，逗号分隔，支持字段值带双引号包裹。

表头示例：
```
isbn,title,author,publisher,quantity
```

数据行示例：
```
9787302546896,Java编程思想,Bruce Eckel,机械工业出版社,3
9787111213826,设计模式,GoF,机械工业出版社,2
```

**各字段校验规则（每行逐字段校验）**

| 字段名 | 必填 | 校验规则 | 行错误原因描述 |
|--------|------|----------|---------------|
| isbn | 是 | 10位或13位纯数字，不含横线或空格 | "isbn格式错误：必须为10位或13位纯数字" |
| title | 是 | 非空，最长100字符 | "title不能为空" / "title超过100个字符" |
| author | 是 | 非空，最长50字符 | "author不能为空" / "author超过50个字符" |
| publisher | 是 | 非空，最长100字符 | "publisher不能为空" / "publisher超过100个字符" |
| quantity | 是 | 正整数，范围1～9999 | "quantity格式错误：必须为1到9999之间的正整数" |

**操作流程**

1. 管理员点击"批量导入"按钮，系统展示文件上传界面。
2. 管理员选择 CSV 文件并点击"开始导入"。
3. 系统校验文件类型和大小：
   - 文件非 `.csv` 扩展名：提示"仅支持 .csv 格式文件"，不上传。
   - 文件超过 10MB：提示"文件大小不能超过10MB"，不上传。
4. 文件通过基础校验后，系统开始解析：
   - 校验表头行，若表头不符合规范，终止导入，提示"CSV 表头格式不正确，应包含列：isbn, title, author, publisher, quantity"。
5. 系统逐行处理数据（跳过表头行）：
   - 对每行进行字段校验，校验失败的行记录行号和错误原因，跳过。
   - 校验通过的行，根据 ISBN 是否存在执行新增或累加库存。
6. 所有行处理完毕后，系统展示导入结果摘要。

**状态说明**

| 状态 | 触发条件 | 系统展示内容 |
|------|----------|-------------|
| 空状态 | 打开批量导入界面，尚未选择文件 | 文件上传区域显示"点击选择或拖拽 CSV 文件到此处"，"开始导入"按钮不可点击 |
| 加载状态 | 文件上传并解析处理中 | 显示进度提示"正在处理，请勿关闭页面..."，"开始导入"按钮不可点击 |
| 成功状态（全部成功） | 所有数据行均处理成功 | 弹出结果摘要："成功新增 N 条，成功更新 N 条，失败 0 条" |
| 部分成功状态 | 部分行成功，部分行失败 | 弹出结果摘要："成功新增 N 条，成功更新 N 条，失败 N 条"，下方列出失败明细（行号 + 原因） |
| 错误状态 | 文件格式错误或表头不符合规范 | 显示对应错误提示，不展示处理结果 |
| 边界状态 | CSV 文件除表头外无数据行 | 提示"CSV 文件中无有效数据行" |

**导入结果摘要格式**

```
导入完成
---------------------------------
成功新增：N 条
成功更新：N 条
失败：N 条

失败明细：
第3行：isbn格式错误：必须为10位或13位纯数字
第7行：quantity格式错误：必须为1到9999之间的正整数
第12行：title不能为空
```

**验收标准**

- [ ] 上传非 `.csv` 文件时，系统拒绝，提示"仅支持 .csv 格式文件"。
- [ ] 上传超过 10MB 的 CSV 文件时，系统拒绝，提示"文件大小不能超过10MB"。
- [ ] CSV 表头列名有任意一列缺失或拼写错误时，导入终止，提示表头格式不正确。
- [ ] 新 ISBN 行（校验通过）：数据库中新增记录，quantity 与 CSV 中一致。
- [ ] 已存在 ISBN 行（校验通过）：数据库中 quantity 累加 CSV 中的值，书名/作者/出版社不变。
- [ ] 同一 CSV 中同一 ISBN 出现多次：每次出现的 quantity 均依次累加。
- [ ] 单行字段校验失败时：该行跳过，其他行正常处理，结果摘要中该行列入失败明细并注明行号和原因。
- [ ] CSV 仅有表头行时，提示"CSV 文件中无有效数据行"。
- [ ] 导入处理中，"开始导入"按钮不可重复点击。

#### 4.1.3 图书查询

**功能概述**

管理员通过关键字搜索图书，支持按书名、作者、ISBN 进行模糊匹配，搜索结果以列表形式分页展示。该功能是管理员执行借还操作前确认图书信息的主要入口。

**用户故事**

As a 图书管理员, I want to 通过书名、作者或 ISBN 搜索图书, So that 快速找到目标图书，确认库存和基本信息。

**业务规则**

| 编号 | 规则 |
|------|------|
| BK-Q-01 | 搜索关键字对书名、作者、ISBN 三个字段进行"包含"匹配（不区分大小写） |
| BK-Q-02 | 搜索关键字为空时，展示全部图书列表 |
| BK-Q-03 | 搜索结果按图书 ID 升序排列 |
| BK-Q-04 | 列表分页，每页显示 20 条 |

**操作流程**

1. 管理员进入图书查询页面，系统展示全部图书列表（分页）。
2. 管理员在搜索框输入关键字，点击"搜索"按钮或按回车键。
3. 系统根据关键字在书名、作者、ISBN 字段中进行模糊匹配，返回结果列表。
4. 管理员可翻页查看更多结果。
5. 管理员清空搜索框并搜索，恢复展示全部图书。

**状态说明**

| 状态 | 触发条件 | 系统展示内容 |
|------|----------|-------------|
| 空状态 | 首次进入图书查询页面，尚未进行搜索 | 展示全部图书列表（按图书ID升序），显示总数"共N本图书" |
| 加载状态 | 搜索请求处理中 | 列表区域显示加载动画，搜索按钮不可点击 |
| 成功状态 | 搜索到匹配结果 | 展示匹配图书列表，显示"共找到N条结果" |
| 错误状态 | 系统查询异常 | 显示提示"查询失败，请刷新页面重试" |
| 边界状态 | 搜索关键字无匹配结果 | 列表区域显示"未找到与'[关键字]'相关的图书" |

**列表展示字段**

| 字段 | 说明 |
|------|------|
| ISBN | 图书 ISBN |
| 书名 | 图书名称 |
| 作者 | 作者姓名 |
| 出版社 | 出版社名称 |
| 库存数量 | 当前可借数量（总量减去已借出未还数量） |
| 操作 | "修改"、"删除"两个操作入口 |

**验收标准**

- [ ] 输入书名关键字，返回书名包含该关键字的所有图书（不区分大小写）。
- [ ] 输入作者关键字，返回作者字段包含该关键字的所有图书。
- [ ] 输入 ISBN 关键字，返回 ISBN 包含该关键字的所有图书。
- [ ] 搜索框为空时，展示全部图书列表。
- [ ] 搜索无结果时，显示"未找到与'[关键字]'相关的图书"，不显示空表格。
- [ ] 列表超过 20 条时，分页控件显示，可正常翻页。

#### 4.1.4 修改图书信息

**功能概述**

管理员对已有图书的书名、作者、出版社、库存数量进行修改。ISBN 字段创建后不可修改（ISBN 是图书的唯一标识）。修改库存数量时，系统允许直接设置总库存数，但总库存数不得小于已借出未还数量。

**用户故事**

As a 图书管理员, I want to 修改图书的基本信息和库存数量, So that 图书信息保持准确，库存数量反映实际馆藏。

**业务规则**

| 编号 | 规则 |
|------|------|
| BK-E-01 | ISBN 字段在修改页面显示但不可编辑 |
| BK-E-02 | 修改库存数量时，新数量必须 ≥ 该图书当前已借出未还数量；若违反则拒绝提交，提示"库存数量不能小于已借出未还数量（当前已借出N本）" |

#### 4.1.5 删除图书

**功能概述**

管理员删除系统中的图书记录。为防止数据不一致，当该图书存在未还借阅记录时，系统禁止删除。删除操作不可撤销，执行前必须展示确认对话框。

**用户故事**

As a 图书管理员, I want to 删除不再需要的图书记录, So that 系统中不保留无效数据。

**业务规则**

| 编号 | 规则 |
|------|------|
| BK-D-01 | 图书存在至少一条"未还"状态的借阅记录时，禁止删除，提示"该图书有N本未归还，无法删除" |
| BK-D-02 | 图书所有借阅记录均已还书（或无借阅记录）时，允许删除 |
| BK-D-03 | 删除操作不可撤销，执行前必须弹出确认对话框，用户点击"确认删除"后方可执行 |
| BK-D-04 | 删除成功后，图书相关的历史借还记录保留，不随图书删除（用于历史查阅） |

**操作流程**

1. 管理员在图书列表中点击某图书的"删除"按钮。
2. 系统检查该图书是否存在未还记录：
   - 存在未还记录：弹出提示"该图书有N本未归还，无法删除"，不展示确认对话框。
   - 无未还记录：弹出确认对话框"确认删除图书《[书名]》？此操作不可撤销。"，显示"确认删除"和"取消"两个按钮。
3. 管理员点击"取消"：关闭对话框，不执行删除。
4. 管理员点击"确认删除"：系统执行删除，从图书列表中移除该图书。
5. 系统提示"删除成功"，刷新图书列表。

**状态说明**

| 状态 | 触发条件 | 系统展示内容 |
|------|----------|-------------|
| 空状态 | 图书列表正常显示，未触发删除 | 每本图书旁有"删除"按钮 |
| 加载状态 | 点击"确认删除"后执行中 | 对话框"确认删除"按钮变为不可点击，显示"删除中..." |
| 成功状态 | 删除成功 | 对话框关闭，图书列表刷新，页面顶部显示"删除成功" |
| 错误状态 | 存在未还记录，不允许删除 | 弹出提示"该图书有N本未归还，无法删除"（N为实际未还数量），不展示删除确认框 |
| 边界状态 | 管理员点击"取消" | 关闭对话框，图书未被删除，列表保持不变 |

**验收标准**

- [ ] 图书有未还借阅记录时，点击"删除"后显示"该图书有N本未归还，无法删除"，数据库记录不变。
- [ ] 图书无未还借阅记录时，点击"删除"弹出确认对话框。
- [ ] 点击"取消"后，图书记录未被删除。
- [ ] 点击"确认删除"后，图书记录从数据库中删除，图书列表中该图书消失。
- [ ] 删除图书后，该图书历史借还记录在数据库中仍存在。

### 4.3 借还管理模块

#### 4.3.1 借书操作

**功能概述**

管理员代表读者执行借书，输入读者 ID 和图书 ISBN，系统完成权限校验（借阅上限、禁借期、库存）后生成借阅记录，并同步扣减图书库存。借出日期为当天操作日期，应还日期为借出日期加30天。

**用户故事**

As a 图书管理员, I want to 通过输入读者ID和图书ISBN完成借书操作, So that 快速为读者办理借书手续，系统自动更新库存和借阅记录。

**业务规则**

| 编号 | 规则 |
|------|------|
| BR-01 | 读者 ID 必须存在于系统，否则拒绝，提示"读者ID不存在，请先注册" |
| BR-02 | 图书 ISBN 必须存在于系统，否则拒绝，提示"图书ISBN不存在，请先录入" |
| BR-03 | 读者当前已借出未还数量 ≥ 该读者最大借书数量时，禁止借书，提示"该读者借阅数量已达上限（上限N本，当前已借N本）" |
| BR-04 | 读者处于禁借期内（禁借截止日期 > 当前操作日期）时，禁止借书，提示"该读者当前处于禁借期，禁借截止日期：YYYY-MM-DD，原因：曾于 YYYY-MM-DD 超期还书" |
| BR-05 | 图书当前库存数量（总量减去已借出未还数量）= 0 时，禁止借出，提示"该图书库存不足，当前库存为0" |
| BR-06 | 同一读者可重复借阅同一本书（已还清的情况下），只要通过上述所有校验 |
| BR-07 | 借出日期 = 当前操作日期（系统服务器日期） |
| BR-08 | 应还日期 = 借出日期 + 30 天（自然日） |
| BR-09 | 借书成功后，图书库存数量（可借数量）减 1 |

**操作流程**

1. 管理员进入借书操作页面，页面显示读者 ID 输入框和图书 ISBN 输入框。
2. 管理员输入读者 ID，系统实时校验读者是否存在（可在输入框失焦时触发）：
   - 读者不存在：在读者 ID 输入框下方显示"读者ID不存在，请先注册"。
   - 读者存在：显示读者信息摘要（姓名、当前借阅数/上限、禁借状态）。
3. 管理员输入图书 ISBN，系统实时校验图书是否存在（可在输入框失焦时触发）：
   - 图书不存在：显示"图书ISBN不存在，请先录入"。
   - 图书存在：显示图书信息摘要（书名、当前库存）。
4. 管理员点击"确认借出"按钮：
   - 系统再次完整执行所有校验（BR-01 ～ BR-05），任一不通过则显示对应错误提示，不生成记录。
   - 所有校验通过：生成借阅记录，图书库存 -1。
5. 系统显示"借书成功"摘要：读者姓名、书名、借出日期、应还日期。管理员确认后可继续下一次借书操作。

**状态说明**

| 状态 | 触发条件 | 系统展示内容 |
|------|----------|-------------|
| 空状态 | 打开借书页面 | 读者ID和图书ISBN输入框均为空，读者信息区和图书信息区显示空白，"确认借出"按钮不可点击 |
| 加载状态 | 点击"确认借出"处理中 | "确认借出"按钮不可点击，显示"处理中..." |
| 成功状态 | 借书记录生成成功 | 显示绿色成功提示：读者姓名、书名、借出日期、应还日期，输入框清空 |
| 错误状态（读者问题） | 读者不存在/达上限/在禁借期 | 在读者信息区显示具体原因文案（见业务规则 BR-01/03/04） |
| 错误状态（图书问题） | 图书不存在/库存为0 | 在图书信息区显示具体原因文案（见业务规则 BR-02/05） |
| 边界状态（禁借期） | 读者在禁借期内 | 读者信息区红色显示："该读者当前处于禁借期，禁借截止日期：YYYY-MM-DD，原因：曾于 YYYY-MM-DD 超期还书" |

**验收标准**

- [ ] 读者 ID 不存在时，点击"确认借出"被拒绝，提示"读者ID不存在，请先注册"。
- [ ] 图书 ISBN 不存在时，点击"确认借出"被拒绝，提示"图书ISBN不存在，请先录入"。
- [ ] 读者当前借阅数 ≥ 借阅上限时，点击"确认借出"被拒绝，提示包含上限值和当前借阅数。
- [ ] 读者处于禁借期内时，点击"确认借出"被拒绝，提示包含禁借截止日期和原因（含超期还书日期）。
- [ ] 图书库存为0时，点击"确认借出"被拒绝，提示"该图书库存不足，当前库存为0"。
- [ ] 所有校验通过时，借阅记录写入数据库，借出日期为当天，应还日期为借出日期 + 30 天。
- [ ] 借书成功后，数据库中该图书可借数量减 1（总库存不变，已借出数 +1）。
- [ ] 借书成功后，成功提示中包含读者姓名、书名、借出日期、应还日期。

#### 4.3.2 还书操作

**功能概述**

管理员代读者执行还书，输入图书 ISBN 和读者 ID（用于确认借阅记录），系统完成还书：更新还书时间、恢复库存，并判断是否超期。若超期，系统对该读者施加 30 天禁借处罚（从还书当天起算，禁借截止日期 = 还书日期 + 30 天）。

**用户故事**

As a 图书管理员, I want to 通过输入图书ISBN和读者ID完成还书操作, So that 快速为读者办理还书手续，系统自动恢复库存并处理超期处罚。

**业务规则**

| 编号 | 规则 |
|------|------|
| RT-01 | 图书 ISBN 必须存在于系统，否则提示"图书ISBN不存在" |
| RT-02 | 读者 ID 必须存在于系统，否则提示"读者ID不存在" |
| RT-03 | 系统在数据库中查找该读者 + 该图书 ISBN 的"未还"状态借阅记录；若不存在，提示"未找到该读者借阅该图书的未还记录" |
| RT-04 | 若同一读者借了多本相同 ISBN 的图书（库存多于1时允许多次借阅），系统选择借出日期最早的那条未还记录进行还书 |
| RT-05 | 还书时，系统将该借阅记录的还书时间设置为当前操作日期 |
| RT-06 | 还书后，图书可借数量 +1 |
| RT-07 | 超期判断：还书日期 > 应还日期，则判定为超期 |
| RT-08 | 超期处罚：若超期，对该读者施加禁借处罚，禁借截止日期 = 还书日期 + 30 天（若读者已有禁借截止日期且晚于新计算值，则保留已有值；否则更新为新计算值） |
| RT-09 | 不超期：不施加任何处罚，不修改读者禁借状态 |

**操作流程**

1. 管理员进入还书操作页面，输入图书 ISBN。
2. 系统实时查询图书是否存在：
   - 不存在：显示"图书ISBN不存在"。
   - 存在：显示图书信息（书名）。
3. 管理员输入读者 ID，系统实时查询读者及对应未还记录：
   - 读者不存在：显示"读者ID不存在"。
   - 读者存在但无该图书的未还记录：显示"未找到该读者借阅该图书的未还记录"。
   - 读者存在且有未还记录：显示借阅信息摘要（借出日期、应还日期、是否已超期）。
4. 管理员点击"确认还书"。
5. 系统执行还书：更新还书时间、图书库存 +1、超期判断与处罚施加。
6. 系统显示还书结果：
   - 不超期：显示"还书成功"。
   - 超期：显示"还书成功，但该图书已超期 N 天，该读者已被施加 30 天禁借处罚（禁借截止：YYYY-MM-DD）"。

**状态说明**

| 状态 | 触发条件 | 系统展示内容 |
|------|----------|-------------|
| 空状态 | 打开还书页面 | 图书ISBN和读者ID输入框均为空，"确认还书"按钮不可点击 |
| 加载状态 | 点击"确认还书"处理中 | "确认还书"按钮不可点击，显示"处理中..." |
| 成功状态（不超期） | 还书成功且未超期 | 绿色提示"还书成功"，显示书名、读者姓名、还书日期 |
| 成功状态（超期） | 还书成功且已超期 | 黄色提示"还书成功，但该图书已超期N天，该读者已被施加30天禁借处罚（禁借截止：YYYY-MM-DD）" |
| 错误状态 | 图书或读者不存在 / 无未还记录 | 在对应输入框下方显示具体原因文案（见业务规则 RT-01/02/03） |
| 边界状态 | 同一读者有多条该图书未还记录 | 系统自动选择借出日期最早的记录还书，结果提示中显示所选记录的借出日期 |

**验收标准**

- [ ] 图书 ISBN 不存在时，"确认还书"被拒绝，提示"图书ISBN不存在"。
- [ ] 读者 ID 不存在时，"确认还书"被拒绝，提示"读者ID不存在"。
- [ ] 读者无该图书的未还记录时，"确认还书"被拒绝，提示"未找到该读者借阅该图书的未还记录"。
- [ ] 还书成功后，数据库中对应借阅记录的还书时间更新为当天日期，状态变为"已还"。
- [ ] 还书成功后，图书可借数量 +1。
- [ ] 还书日期 ≤ 应还日期时，不施加禁借处罚，读者禁借状态不变。
- [ ] 还书日期 > 应还日期时，读者禁借截止日期更新为还书日期 + 30 天（若原禁借截止日期更晚，则保留原值），页面提示超期天数和新禁借截止日期。
- [ ] 同一读者有同一图书多条未还记录时，系统选择借出日期最早的记录执行还书。

#### 4.3.3 逾期列表

**功能概述**

管理员查看当前所有逾期未还的借阅记录，即当前日期已超过应还日期且尚未还书的记录。列表提供实时视图，方便管理员掌握逾期情况，督促读者归还。

**用户故事**

As a 图书管理员, I want to 一键查看所有当前逾期未还的借阅记录, So that 及时跟进逾期读者，督促还书。

**业务规则**

| 编号 | 规则 |
|------|------|
| OD-01 | 逾期定义：当前日期（服务器日期） > 应还日期，且该借阅记录还书时间为空（未还） |
| OD-02 | 已逾期天数 = 当前日期 - 应还日期（自然日，当前日期等于应还日期时为0天，不计为逾期） |
| OD-03 | 列表按已逾期天数降序排列（逾期最久的排在最前） |
| OD-04 | 列表无分页，展示全部逾期记录（假定逾期记录数量在合理范围内，不超过1000条） |
| OD-05 | 列表数据为实时查询，每次打开或刷新页面时重新计算 |

**操作流程**

1. 管理员点击"逾期列表"菜单，系统查询所有满足逾期条件的借阅记录。
2. 系统展示逾期列表，按已逾期天数降序排列。
3. 管理员可刷新页面以获取最新数据。

**状态说明**

| 状态 | 触发条件 | 系统展示内容 |
|------|----------|-------------|
| 空状态 | 当前无任何逾期记录 | 显示"当前无逾期记录" |
| 加载状态 | 页面数据加载中 | 列表区域显示加载动画 |
| 成功状态 | 查询到逾期记录 | 显示逾期列表，标注"共N条逾期记录" |
| 错误状态 | 系统查询异常 | 显示"查询失败，请刷新页面重试" |
| 边界状态 | 某条记录的应还日期恰好为今天 | 该记录不计为逾期，不出现在逾期列表中（已逾期天数须 > 0 才显示） |

**逾期列表展示字段**

| 字段 | 说明 |
|------|------|
| 读者ID | 借阅该图书的读者唯一标识 |
| 读者姓名 | 读者姓名 |
| 图书ISBN | 逾期图书的ISBN |
| 图书名称 | 逾期图书的书名 |
| 借出日期 | 该借阅记录的借出日期（格式：YYYY-MM-DD） |
| 应还日期 | 该借阅记录的应还日期（格式：YYYY-MM-DD） |
| 已逾期天数 | 当前日期 - 应还日期（正整数，单位：天） |

**验收标准**

- [ ] 逾期列表仅展示"当前日期 > 应还日期 且 还书时间为空"的借阅记录。
- [ ] 应还日期等于当前日期的记录不出现在逾期列表中。
- [ ] 逾期列表按已逾期天数降序排列（最久的在最上方）。
- [ ] 已逾期天数 = 当前日期 - 应还日期，计算值与字段展示值一致。
- [ ] 某记录还书后，刷新逾期列表，该记录不再出现。
- [ ] 无逾期记录时，显示"当前无逾期记录"，不显示空表格。

## 六、数据需求

### 6.1 核心数据实体

**图书（Book）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | 整数，主键，自增 | 系统内部图书ID |
| isbn | 字符串，唯一，非空 | ISBN，10位或13位数字 |
| title | 字符串，非空 | 书名，最长100字符 |
| author | 字符串，非空 | 作者，最长50字符 |
| publisher | 字符串，非空 | 出版社，最长100字符 |
| total_quantity | 整数，非空 | 总库存数量（含借出未还） |
| created_at | 日期时间 | 录入时间 |
| updated_at | 日期时间 | 最后修改时间 |

说明：可借数量 = total_quantity - 该图书未还借阅记录数，该值为计算值，不单独存储。

**读者（Reader）**

| 字段 | 类型 | 说明 |
|------|------|------|
| reader_id | 字符串，主键 | 读者唯一标识（由管理员录入） |
| name | 字符串，非空 | 读者姓名，最长30字符 |
| contact | 字符串，可空 | 联系方式（11位手机号） |
| max_borrow_count | 整数，非空，默认5 | 最大同时借阅数量，范围1～20 |
| ban_until | 日期，可空 | 禁借截止日期；为空或早于当前日期表示无禁借限制 |
| ban_reason_date | 日期，可空 | 导致最近一次禁借的超期还书日期（用于展示禁借原因） |
| created_at | 日期时间 | 注册时间 |
| updated_at | 日期时间 | 最后修改时间 |

**借阅记录（BorrowRecord）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | 整数，主键，自增 | 系统内部记录ID |
| reader_id | 字符串，外键 → Reader | 借阅读者 |
| book_isbn | 字符串，外键 → Book.isbn | 借阅图书 |
| borrow_date | 日期，非空 | 借出日期 |
| due_date | 日期，非空 | 应还日期（= borrow_date + 30天） |
| return_date | 日期，可空 | 实际还书日期；为空表示未还 |
| created_at | 日期时间 | 记录创建时间 |

### 6.2 数据规则

| 规则 | 说明 |
|------|------|
| 可借数量计算 | 实时计算：total_quantity - COUNT(BorrowRecord WHERE book_isbn = X AND return_date IS NULL) |
| 当前借阅数量 | 实时计算：COUNT(BorrowRecord WHERE reader_id = X AND return_date IS NULL) |
| 禁借判断 | Reader.ban_until IS NOT NULL AND Reader.ban_until > 当前日期 |
| 逾期判断 | BorrowRecord WHERE return_date IS NULL AND due_date < 当前日期 |
| 借书事务原子性 | 插入 BorrowRecord + 不单独更新库存（库存为计算值）在同一事务中校验并提交 |
| 还书事务原子性 | 更新 BorrowRecord.return_date + 可能的 Reader.ban_until 更新 在同一事务中执行 |

## 七、术语表

| 术语 | 定义 |
|------|------|
| ISBN | 国际标准书号（International Standard Book Number），图书的唯一编号，本系统支持10位或13位纯数字格式 |
| 借出日期 | 管理员执行借书操作时的服务器日期（不含时间，精确到天） |
| 应还日期 | 借出日期 + 30 自然日，是读者被要求归还图书的最晚日期 |
| 还书日期 | 管理员执行还书操作时的服务器日期（不含时间，精确到天） |
| 逾期 | 当前日期严格大于应还日期，且借阅记录未还（return_date 为空）的状态 |
| 已逾期天数 | 当前日期 - 应还日期，结果为正整数（仅在逾期状态下计算） |
| 禁借期 | 读者因超期还书被施加的处罚期间，期间内该读者不能借阅任何图书 |
| 禁借截止日期 | 禁借期结束的日期，当前日期超过该日期后，禁借自动解除（下次借书时系统重新校验） |
| 库存数量（总量） | 该图书在图书馆的总馆藏册数（含借出未还的） |
| 可借数量 | 库存数量（总量）减去当前已借出未还数量，表示目前可以被借出的册数 |
| 未还记录 | BorrowRecord.return_date 为空的借阅记录 |
| 读者ID | 管理员手动录入的读者唯一标识符，通常对应图书馆读者证编号 |
